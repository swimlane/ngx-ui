Welcome to Swim-UI

Lit web component library matching Swimlane's ngx-ui design system. Use the sidebar to explore components.

Load swim-ui from the Swimlane CDN. You can load the full bundle or individual components; version locking is supported for predictable builds.

Individual components (recommended)

Import only the components you use. This keeps payload size small—you avoid downloading the rest of the library. Each component script is self-contained (includes Lit), so no import map is required. Use the same base URL and version path as the full bundle; append the component name (e.g. button.js, input.js). 

// Only <swim-button> and <swim-input> are registered — smaller download
import 'https://widgets.swimlane.app/swim-ui/button.js';
import 'https://widgets.swimlane.app/swim-ui/input.js';

...
<swim-button variant="primary">Click Me</swim-button>
<swim-input label="Name" placeholder="Enter name"></swim-input>

Available component scripts include button.js, input.js, select.js, card.js, dialog.js, tabs.js, checkbox.js, toggle.js, and others. Version locking works the same way (e.g. .../1.0/button.js).

Full bundle

One script registers all custom elements. Use this when you need many components; otherwise prefer individual component imports for a smaller download. 


import 'https://widgets.swimlane.app/swim-ui/swim-ui.js';
...
<swim-button variant="primary">Click Me</swim-button>

Version locking

Pin the version in the URL to avoid unexpected updates. You can lock to major, minor, or patch (e.g. 1, 1.0, or 1.0.0).

  • Latest (no version) — always gets the newest release:
    https://widgets.swimlane.app/swim-ui/swim-ui.js

  • Major (e.g. 1) — any 1.x.x (minor and patch updates):
    https://widgets.swimlane.app/swim-ui/1/swim-ui.js

  • Minor (e.g. 1.0) — any 1.0.x (patch updates only):
    https://widgets.swimlane.app/swim-ui/1.0/swim-ui.js

  • Patch (e.g. 1.0.0) — exact version, no automatic updates:
    https://widgets.swimlane.app/swim-ui/1.0.0/swim-ui.js
