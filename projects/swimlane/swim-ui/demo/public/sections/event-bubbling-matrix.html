Event propagation matrix

Use this page to see where change / checked-change from swim-checkbox are observed. swim-ui dispatches these with bubbles: false and composed: false, so they stay on the host: attach listeners on the swim-checkbox (or a direct listener target), not on document, and not from outside an enclosing shadow root unless you listen on the host inside that root.

Case A – light DOM

Wrapper .matrix-wrap-light → swim-checkbox. Listeners: checkbox, wrapper (bubble), document (bubble).

Case B – swim-checkbox inside another shadow root

The checkbox lives in an element's shadow root. With non-bubbling events, a listener on document does not receive change from inside the shadow subtree; listen on the checkbox node or the shadow host.
