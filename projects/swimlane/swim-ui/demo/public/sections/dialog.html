// Optional: myDialog.closeOnBlur = false; // or attribute close-on-blur="false"
// Optional: myDialog.closeOnEscape = false;

swim-dialog dispatches open when visible becomes true and close when it becomes false. Host events use bubbles: false and composed: false, so attach listeners on the swim-dialog element itself, not on a parent for delegation. For large/medium layout, swim-large-format-dialog-content dispatches close-or-cancel (detail: dirty boolean) from the header control — listen on that element, then set the parent dialog’s visible to false if you want the overlay to dismiss.

Properties

AttributeTypeDefaultDescription
dialog-titlestring''Title text in the dialog header
format'regular' | 'medium' | 'large''regular'Dialog size format
close-buttonbooleantrueWhen false, hides the regular-format X and the large/medium header close/cancel (dismiss via backdrop, Escape, or hide() unless disabled)
close-on-blurbooleantrueWhen false, backdrop click does not close (ngx closeOnBlur)
close-on-escapebooleantrueWhen false, Escape does not close (ngx closeOnEscape)
visiblebooleanfalseControls visibility (or call show()/hide())
z-indexnumber991CSS z-index stacking order

Events

EventDetailDescription
open—Fired when the dialog is shown; host-only (does not bubble)
close—Fired when the dialog is closed; host-only (does not bubble)
