Dialog - Large Format

Large Format Dialog is a regular Dialog whose content is wrapped inside of NgxLargeFormatDialogContent component

Large Format Dialog has the following styles:

  • Top/bottom spacing: 50px
  • Left/Right spacing: 60px
  • Header height: 90px
  • Footer height: 64px
  • Body Left/Right spacing: 32px each side

There are two requirements to use Large Format Dialog:

  • Wrap your content inside of ngx-large-format-dialog-content
  • Provide format: DialogFormat.Large to the options object when invoking DialogService#create

Available CSS variables are as followed:

  • --header-item-max-width: Max width of Header Title and Header Action sections (default to 400px)
  • --header-item-max-width-lg: Max width of Header Title and Header Action sections below lg breakpoint (default to 250px)
  • --header-item-max-width-md: Max width of Header Title and Header Action sections below md breakpoint (default to 150px)

For Stepper variant:

  • --stepper-header-custom-offset: Additional white space between the steps to the Header Title and Header Action sections (default to 0px)

For Tabs variant:

  • --tabs-header-gap: The spacing between the tabs' labels (default to 1.25rem)
For --tabs-header-gap, the consumers need to handle responsiveness (different values for --tabs-header-gap at different breakpoints) on their own.

ngx-large-format-dialog-footer will always be at the bottom of the Dialog with the max height of 64px

ngx-large-format-dialog-footer applies some default styles to set the children to the center with a default gap of 0.5rem. The gap can be customized by using --item-gap CSS variable

You can pass in [styleClass] to completely nullify the default styles.

The semantic of the Action Button is that it has a concept of dirty state

In dirty state, the Action Button will mean "Cancelling out everything and close the Dialog". While in clean state, the Action Button will mean "Just close the dialog".

Action Button text will change based on this state. By default, Close for clean state and Cancel for dirty state. The text can be customized via Input. However, the consumers need to determine what to do when, upon invoking Action Button, the state is dirty or clean

When the users attempt to close the Dialog while in dirty state, LargeFormatDialogContent will display a Confirmation on close to ensure the users indeed want to close the dialog and discard their changes.

You can pass in [dirtyAlertOptions] to customize the content of the Confirmation

You can also skip the Confirmation altogether by setting [skipDirtyAlert] to true. Try toggling the button below then open the Dialog again

ngx-large-format-dialog-content will be switched to Stepper Variant when it finds a LargeFormatDialogStepperDirective instance. If there is a ngx-stepper, custom styles will be applied to this Stepper.
