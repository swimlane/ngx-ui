Component Inputs

Name Description
@Input() clickable: boolean Whether the component allows clicking. If enabled, the date/time will be copied to the user's clipboard.
@Input() clipFormat: string NEEDS DOCUMENTATION.
@Input() datetime: Datelike = new Date() The date/time for the component to display.
@Input() defaultCopyKey: string = 'Local' NEEDS DOCUMENTATION.
@Input() defaultInputTimeZone: string The default timezone for the component to display times in. Overrides the timezone Input.
@Input() format: string The format to display the provided Date object in. See Examples tab.
@Input() invalidDateMessage: string = 'Invalid date' The message displayed if the provided Date object is invalid.
@Input() mode: DATE_DISPLAY_TYPES The mode for the component to display dates and times. See Examples tab.
@Input() precision: moment.unitOfTime.StartOf Mutates the original moment by setting it to the start of a unit of time.
@Input() timezone: string = moment.tz.guess() The timezone for the component to display times in.
@Input() timezones: Record<string, string> = { UTC: 'Etc/UTC', Local: '' } Used to display dates in other timezones.
@Input() tooltipCssClass: string = 'date-tip-tooltip' The CSS class for styling the component tooltip.
@Input() tooltipDisabled: boolean Whether the component tooltip is disabled.
@Input() tooltipFormat: string The format to display the provided Date object in on the component tooltip.
@Input() tooltipPlacement: string = 'top' The placement of the component tooltip.
@Input() tooltipTemplate: string The template to render in the component tooltip.
@Input() type: string The type of the component e.g. date, time, dateTime. See Examples tab.
