ngx-columns Column

Name Description
@Input() column: Partial<Column>

Single Column that either displays a list of options for the user to interact with or custom content. This column should include the entire nested structure of the columns and should have the `active` property set to `true`. Below is an example. 

{{ columnExample | json }}

Notice how each Column requires a unique identifier (id), active property, title which displays the label when the Column is in a list or when active, the name of the adjacent Column.

The first Column should always have an active property set to true to be displayed correctly. Any Column you wish to open on load that is descendant to this Column should also have it's active property set to true.

Column can either accept a list of Columns set on the children property or a custom template set on the content property. Internally ngx-columns will display either a list of options or the custom content specified. The content property model maps to the same interface used by *ngComponentOutlet. When content is specified, use component, inputs, and outputs to pass models to the internal view.

width is an additional property on content that allows you to specify a custom width for the content area, otherwise the ColumnsComponent will try to auto layout the content area to fill the remaining width.

In the above example, the component is referenced as a string for visibility. In practice, pass the reference to the Component you want to display in the custom content area.
